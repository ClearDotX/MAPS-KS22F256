void DSPI_MasterCreateEdmaHandle ( SPI_Type *  base,
dspi_master_edma_handle_t *  handle,
dspi_master_edma_transfer_callback_t  callback,
void *  userData,
edma_handle_t *  edmaRxRegToRxDataHandle,

edma_handle_t *  edmaIntermediaryToTxRegHandle 
)

This function initializes the DSPI edma handle which can be used for other DSPI transactional APIs. Usually, for a specified DSPI instance, user need only call this API once to get the initialized handle.

Note that DSPI edma has separated(rx and tx , 2 source) or shared(rx and tx is the same source) dma request source . (1)For separated dma request source , you should enable and set the Rx DMAMUX source for edmaRxRegToRxDataHandle and Tx DMAMUX source for edmaIntermediaryToTxRegHandle. (2)For shared dma request source , you should enable and set the Rx/Rx DMAMUX source for edmaRxRegToRxDataHandle .
