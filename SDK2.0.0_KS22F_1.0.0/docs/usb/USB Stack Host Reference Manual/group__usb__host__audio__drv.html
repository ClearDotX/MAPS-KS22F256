USB AUDIO Class driver
USB Class driver

Overview

The audio device class definition applies to all devices or functions embedded in composite devices that are used to manipulate audio, voice, and sound-related functionality. This includes both audio data (analog and digital) and the functionality that is used to directly control the audio environment, such as volume and tone Control. Typical examples of audio class devices: USB audio speaker. This section describes the programming interface of the USB HOST audio class driver. The USB HOST audio class driver handles the specific control requests for audio class, transfers data from and to the device through the isochronous pipe.

USB Host audio Initialization

When audio device is attached, audio initialization flow works as follow:

The above picture describes the following steps:

  • call USB_HostAudioInit to initialize audio class instance audio_instance_t, the return class handle pointer to the audio class instance. The driver uses an instantiation of the audio_instance_t structure to maintain the current state of a audio instance module driver. This structure holds the usb host handle, usb device handle and keeps track of transfer information, alternate setting, pipes and interfaces that are enumerated for attached audio device.
  • call USB_HostAudioControlSetInterface to set the audio class control interface. this interface's pipes will be opened.
  • wait last step operation callback.
  • call USB_HostAudioStreamSetInterface to set the audio class stream interface. this interface's pipes will be opened.
  • wait last step operation callback.
  • call USB_HostAudioStreamRecv to receive isochronous data from device, or call USB_HostAudioStreamSend to send isochronous data to device.
  • wait last step operation callback.
  • process data and receive or send again.

USB Host audio De-initialization

application can call usb_host_audio_deinit to de-initialize audio. This function cancel transfer, close pipe and release audio class instance.
There are two cases to call this function:

  • audio device is detached: this function is called to release resource.
  • application call this function, then call USB_HostAudioInit to re-initialize audio class.

USB Host audio Send data

Provide buffer pointer, buffer length, callback function and callback parameter and call USB_HostAudioStreamSend to start asynchronous sending. Then callback function is called with one transfer status parameter when the transfer succeed or fail.

USB Host audio Receive data

Provide buffer pointer, buffer length, callback function and callback parameter and call USB_HostAudioStreamRecv to start asynchronous receiving. Then callback function is called with one transfer status parameter when the transfer succeed or fail.
