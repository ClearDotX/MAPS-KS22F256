USB CDC Class driver
USB Class driver

Overview

The USB Communications Device Class (or USB CDC) The Communications Class defines mechanisms for a device and host to identify which existing protocols to use. it defines an architecture that is capable of supporting any communications device. Communications Device Class and associated subclass specifications,such as ISDN,PSTN, provides information to guide implementers in using the USB logical structures for communications devices. This section use PSTN as the subclass and describes the programming interface of the USB HOST cdc class driver. The USB HOST HID class driver handles the specific control requests for CDC class, transfers data from and to the device through the bulk pipe.

USB Host CDC Initialization

When cdc device is attached, cdc initialization flow works as follow:

The above picture describes the following steps:

  • call USB_HostCdcInit to initialize cdc class instance #cdc_instance_struct_t, save class handle pointer into the #cdc_instance_struct_t. The driver uses an instantiation of the cdc_instance_struct_t structure to maintain the current state of a CDC instance module driver. This structure holds the usb host handle, usb device handle and keeps track of transfer information, alternate setting, pipes and interfaces that are enumerated for attached CDC device.
  • call USB_HostCdcSetControlInterface to set the cdc class control interface. this interface's pipes will be opened.
  • wait last step operation callback.
  • call USB_HostCdcSetDataInterface to set the cdc class data interface. this interface's pipes will be opened.
  • wait last step operation callback.
  • call USB_HostCdcDataRecv to receive data from deivice, or call USB_HostCdcDataSend to send data to device.
  • wait last step operation callback.
  • process data and receive or send again.

USB Host CDC De-initialization

application can call USB_HostCdcDeinit to de-initialize cdc. This function cancel transfer, close pipe and release hid class instance.
There are two cases to call this function:

  • cdc device is detached: this function is called to release resource.
  • application call this function, then call USB_HostCdcInit to re-initialize cdc class.

USB Host CDC Send data

Provide buffer pointer, buffer length, callback function and callback parameter and call USB_HostCdcDataSend to start asynchronous sending. Then callback function is called with one trasnfer status parameter when the transfer succeed or fail.

USB Host CDC Receive data

Provide buffer pointer, buffer length, callback function and callback parameter and call USB_HostCdcDataRecv to start asynchronous receiving. Then callback function is called with one trasnfer status parameter when the transfer succeed or fail.
