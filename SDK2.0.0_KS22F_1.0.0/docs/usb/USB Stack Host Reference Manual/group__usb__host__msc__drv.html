USB MSC Class driver
USB Class driver

Overview

The USB Mass Storage Class (or USB MSC) defines mass storage usb device. Typical examples are: U-disk. This section describes the programming interface of the USB Host MSC class driver. The USB Host MSC class driver handles the specific control requests for MSC class, transfers data from and to the device through the interrupt pipe.

USB Host MSC Initialization

When msd device is attached, msd initialization flow works as follow:

The above picture describes the following steps:

  • call USB_HostMsdInit to initialize msd class instance #msd_instance_t, the return class handle pointer to the msd class instance. The driver uses an instantiation of the msd_instance_t structure to maintain the current state of a MSC instance module driver. This structure holds the usb host handle, usb device handle and keeps track of transfer information, alternate setting, pipes and interfaces that are enumerated for attached MSC device.
  • call USB_HostMsdSetInterface to set the msd class interface. this interface's pipes will be opened.
  • wait last step operation callback.
  • test msd device: read capacity, write data or read data.

USB Host MSC De-initialization

application can call USB_HostMsdDeinit to de-initialize msd. This function cancel transfer, close pipe and release msd class instance.
There are two cases to call this function:

  • msd device is detached: this function is called to release resource.
  • application call this function, then call USB_HostMsdInit to re-initialize msd class.

USB Host MSC UFI Command

Provide buffer pointer, buffer length, callback function, callback parameter and other parameters and call USB_HostMsdxx to start asynchronous msd ufi command. Then callback function is called with one command status parameter when the command succeed or fail. 

for example:
USB_HostMsdRead10 need parameters: buffer pointer, reading length, reading block number, callback function, callback parameter, logical unit number and start block address.

Field Documentation

uint32_t usb_host_cbw_t::CBWSignature

The signature field shall contain the value 43425355h (little endian), indicating a CBW

uint32_t usb_host_cbw_t::CBWTag

The device shall echo the contents of this field back to the host in the dCSWTag field of the associated CSW

uint8_t usb_host_cbw_t::CBWFlags

Bit 6 Obsolete. The host shall set this bit to zero. Bits 5..0 Reserved - the host shall set these bits to zero.

uint8_t usb_host_cbw_t::CBWCBLength

This defines the valid length of the command block. The only legal values are 1 through 16 (01h through 10h).
