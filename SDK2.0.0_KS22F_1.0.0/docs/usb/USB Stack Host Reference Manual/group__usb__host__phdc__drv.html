USB PHDC Class driver
USB Class driver

Overview

The USB Personal Healthcare Device Class (or USB PHDC) defines personal healthcare device such as weight scale, thermometer, blood pressure, glucose meters and pulse oximeters. This section describes the programming interface of the USB Host PHDC class driver. The USB Host PHDC class driver handles the specific control requests for PHDC class, transfers data from and to the device through the interrupt and bulk pipes.

USB Host PHDC Initialization

When personal healthcare device is attached, phdc initialization flow works as follow:

The above picture describes the following steps:

  • call USB_HostPhdcInit to initialize phdc class instance usb_host_phdc_instance_t, the return class handle pointer to the phdc class instance. The driver uses an instantiation of the usb_host_phdc_instance_t structure to maintain the current state of a PHDC instance module driver. This structure holds the usb host handle, usb device handle and keeps track of transfer information, alternate setting, pipes and interfaces that are enumerated for attached personal healthcare device.
  • call USB_HostPhdcSetInterface to set the phdc class interface. this interface's pipes will be opened.
  • wait last step operation callback.
  • call USB_HostPhdcRecv to receive data from device, or call USB_HostPhdcSend to send data to device.
  • wait last step operation callback.
  • process data and receive or send again.

USB Host PHDC De-initialization

application can call usb_host_phdc_deinit to de-initialize phdc. This function cancels transfer, closes pipe and releases phdc class instance.
There are two cases to call this function:

  • personal healthcare device is detached: this function is called to release resource.
  • application calls this function, then calls USBHostPhdcInit to re-initialize phdc class.

USB Host PHDC Send data

Provides buffer pointer, buffer length, callback function, callback parameter and call USB_HostPhdcSend function to start asynchronous sending. Then callback function is called with one transfer status parameter when the transfer succeed or fail.

USB Host PHDC Receive data

Provides buffer pointer, buffer length, callback function, callback parameter and call USB_HostPhdcRecv function to start asynchronous receiving. Then callback function is called with one transfer status parameter when the transfer succeed or fail.
