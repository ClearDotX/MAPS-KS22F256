USB Host driver

Overview

The USB host driver implements usb host basic functions: device enumeration, usb standard request, send/receive data. It is the middle layer between class driver and controller driver. It provides same APIs for different controller drivers.

typedef usb_status_t(* host_callback_t)(usb_device_handle deviceHandle, usb_host_configuration_handle configurationHandle, uint32_t eventCode)

This callback function is used to notify application device attach/detach event. This callback pointer is passed when initialize host.

Parameters
deviceHandleThe device handle, it means the attached device.
configurationHandleThe configuration handle, it contains the attached device's configuration information.
event_codeThe callback event code, please refer to the enumeration host_event_t.
Returns
A USB error code or kStatus_USB_Success.
Return values
kStatus_USB_SuccessApplication handles the attached device successfully.
kStatus_USB_NotSupportedApplication don't support the attached device.
kStatus_USB_ErrorApplication handles the attached device falsely.

Parameters

dataThe data buffer pointer.

statusA USB error code or kStatus_USB_Success.
