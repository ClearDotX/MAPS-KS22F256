USB HID Class driver
USB Class driver

Overview

The USB Human Interface Devices (or USB HID) consists primarily of devices that are used by humans to control the operation of computer systems. Typical examples of HID class devices: keyboard and mouse. This section describes the programming interface of the USB HOST HID class driver. The USB HOST HID class driver handles the specific control requests for HID class, transfers data from and to the device through the interrupt pipe.

USB Host HID Initialization

When hid device is attached, hid initialization flow works as follow:

The above picture describes the following steps:

  • call USB_HostHidInit to initialize hid class instance #hid_instance_t, the return class handle pointer to the hid class instance. The driver uses an instantiation of the hid_instance_t structure to maintain the current state of a HID instance module driver. This structure holds the usb host handle, usb device handle and keeps track of transfer information, alternate setting, pipes and interfaces that are enumerated for attached HID device.
  • call USB_HostHidSetInterface to set the hid class interface. this interface's pipes will be opened.
  • wait last step operation callback.
  • call USB_HostHidSetIdle to set hid device.
  • wait last step operation callback.
  • call USB_HostHidGetReportDescriptor to get hid report descriptor.
  • wait last step operation callback.
  • call USB_HostHidSetProtocol to set protocol.
  • wait last step operation callback.
  • call USB_HostHidRecv to receive data from deivice, or call USB_HostHidSend to send data to device.
  • wait last step operation callback.
  • process data and receive or send again.

USB Host HID De-initialization

application can call USB_HostHidDeinit to de-initialize hid. This function cancel transfer, close pipe and release hid class instance.
There are two cases to call this function:

  • hid device is detached: this function is called to release resource.
  • application call this function, then call USB_HostHidInit to re-initialize hid class.

USB Host HID Send data

Provide buffer pointer, buffer length, callback function and callback parameter and call USB_HostHidSend to start asynchronous sending. Then callback function is called with one trasnfer status parameter when the transfer succeed or fail.

USB Host HID Receive data

Provide buffer pointer, buffer length, callback function and callback parameter and call USB_HostHidRecv to start asynchronous receiving. Then callback function is called with one trasnfer status parameter when the transfer succeed or fail.
