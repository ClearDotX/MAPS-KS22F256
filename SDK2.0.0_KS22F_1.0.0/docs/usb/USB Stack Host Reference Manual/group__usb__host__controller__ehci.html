Macro Definition Documentation

#define USB_HOST_EHCI_ISO_BOUNCE_FRAME_NUMBER   (2U)

There is an interval when application send two FS/LS iso transfer. when the interval is less than the macro the two transfer is continuous in the frame list, otherwise the two transfer will not be continuous. for examples:

  • case 1: when inserting sitd firstly, the inserted frame = the current frame value + this MACRO value.
  • case 2: when the inserting sitd is not first, judge the interval between the last inserted frame vale with the current frame value. If the interval is less than this MACRO value, the new sitd is continous with the last sitd. If not, the new sitd inserting frame = the current frame value + this MACRO value.
#define USB_HOST_EHCI_ISO_BOUNCE_UFRAME_NUMBER   (16U)

There is an interval when application send two HS iso transfer. when the interval is less than the macro the two transfer is continuous in the frame list, otherwise the two transfer will not be continuous. for examples:

  • case 1: when inserting itd firstly, the inserted micro-frame = the current micro-frame value + this MACRO value.
  • case 2: when the inserting itd is not first, judge the interval between the last inserted micro-frame vale with the current micro-frame value. If the interval is less than this MACRO value, the new itd is continous with the last itd. If not, the new itd inserting micro-frame = the current micro-frame value + this MACRO value.
